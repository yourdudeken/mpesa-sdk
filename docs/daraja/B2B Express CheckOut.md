# B2B Express CheckOut API

Initiates B2B payment transactions between businesses.

**Endpoint:** `POST https://sandbox.safaricom.co.ke/mpesa/b2bexpressckeckout/v1/paymentrequest`

## Overview
The B2B Express Checkout API enables businesses to initiate payment requests to other businesses. This is used for inter-business transactions including supplier payments, vendor settlements, and bulk business transfers.

### Key Features
- Business-to-Business payment initiation
- Asynchronous transaction processing
- Real-time payment status notifications
- Secure OAuth 2.0 authentication

## How It Works
1. Merchant initiates payment request to receiver business
2. API validates and sends acknowledgment
3. Receiver business receives payment notification
4. Receiver accepts or rejects payment
5. Results sent to initiator via callback URL
6. Both parties receive transaction confirmation

## Use Cases
- Supplier payments between businesses
- Vendor settlement payments
- Inter-company fund transfers
- Business-to-business invoicing
- Bulk B2B payment processing

## Request Body
```json
{
  "InitiatorName": "testuser",
  "SecurityCredential": "SAFVNChNHfVtXEZMBuVo+a1Hwr+DtrUVN3zVg==",
  "CommandID": "DisburseFundsToBusinesses",
  "SenderIdentifierType": "4",
  "RecieverIdentifierType": "2",
  "Amount": "1000",
  "PartyA": "600000",
  "PartyB": "600001",
  "AccountReference": "ref123",
  "Remarks": "Business payment",
  "QueueTimeOutURL": "http://myservice:8080/queuetimeouturl",
  "ResultURL": "http://myservice:8080/result"
}
```

## Request Parameter Definition

| Name | Description | Type | Sample |
|------|-------------|------|--------|
| InitiatorName | Initiator username for authentication | Alpha-Numeric | testapi772 |
| SecurityCredential | Encrypted initiator password | String | EToK4lNR... |
| CommandID | B2B command (DisburseFundsToBusinesses, TransferFundsBetweenAccounts, MerchantToMerchantTransfer) | String | DisburseFundsToBusinesses |
| SenderIdentifierType | Type of sender organization (1: MSISDN, 2: Till, 4: Shortcode) | Numeric | 4 |
| RecieverIdentifierType | Type of receiver organization (1: MSISDN, 2: Till, 4: Shortcode) | Numeric | 2 |
| Amount | Transaction amount | Numeric | 1000 |
| PartyA | Sender shortcode/till | Numeric | 600000 |
| PartyB | Receiver shortcode/till | Numeric | 600001 |
| AccountReference | Alpha-numeric identifier for reference | Alpha-Numeric | ref123 |
| Remarks | Comments sent with transaction | String | B2B payment |
| QueueTimeoutURL | Endpoint for timeout notification | URL | https://ip:port/path |
| ResultURL | Destination URL for result notification | URL | https://ip:port/path |

## Response Body
```json
{
  "OriginatorConversationID": "515-5258779-3",
  "ConversationID": "AG_20200123_0000417fed8ed666e976",
  "ResponseCode": "0",
  "ResponseDescription": "Accept the service request successfully"
}
```

## Response Parameter Definition

| Name | Description | Type | Sample |
|------|-------------|------|--------|
| OriginatorConversationID | Unique identifier of request message | String | 515-5258779-3 |
| ConversationID | Unique identifier generated by M-Pesa | String | AG_20200123_0000417fed8ed666e976 |
| ResponseCode | Indicates if request accepted (0=success) | String | 0 |
| ResponseDescription | Description of Response Code | String | Accept the service request successfully |

## Error Codes

| errorCode | errorMessage | Mitigation | HTTP |
|-----------|-------------|------------|------|
| 400.002.02 | Bad Request - Invalid XXXX | Ensure payload is correct | 400 |
| 401.002.01 | Invalid Access Token | Regenerate new access token | 401 |
| 404.002.01 | Resource not found | Check API endpoint | 404 |
| 405.001 | Method Not Allowed | Use POST method | 405 |
| 500.001.1001 | Merchant does not exist | Use correct PartyA/PartyB | 500 |
| 500.003.02 | Spike Arrest Violation | Reduce request rate | 500 |
| 500.003.03 | Quota Violation | Reduce request rate | 500 |
