# B2C Account Top Up API

Enables businesses to top up customer M-Pesa accounts.

**Endpoint:** `POST https://sandbox.safaricom.co.ke/mpesa/b2c/v3/paymentrequest`

## Overview
The B2C Account Top Up API allows businesses and service providers to automatically credit customer M-Pesa accounts. This is commonly used for telecom airtime top-ups, utility payments, and account balance additions.

### Key Features
- Automated account credit/top-up
- Multiple transaction types support
- Real-time balance updates
- Asynchronous processing with callbacks
- Secure encrypted transactions

## How It Works
1. Business submits top-up request with customer phone number and amount
2. API validates and forwards to M-Pesa
3. M-Pesa credits customer account immediately
4. Customer receives SMS notification
5. Response sent to business via callback URL
6. Transaction logs available for reconciliation

## Use Cases
- Mobile phone airtime top-ups
- Prepaid account credits
- Service balance additions
- Utility bill payments
- Subscription renewals
- Promotional balance credits

## Getting Started

### Prerequisites
- Daraja Account on Safaricom Developer Portal
- Sandbox app with API credentials
- Consumer Key & Consumer Secret
- B2C merchant account configured
- Live till number or shortcode for production

### Good to Know
This API is asynchronous. Responses sent via callback URL with transaction results.

## Request Body
```json
{
  "InitiatorName": "testuser",
  "SecurityCredential": "SAFVNChNHfVtXEZMBuVo+a1Hwr+DtrUVN3zVg==",
  "CommandID": "SalaryPayment",
  "Amount": "500",
  "PartyA": "600000",
  "PartyB": "254722000000",
  "Remarks": "Account top up",
  "QueueTimeOutURL": "http://myservice:8080/queuetimeouturl",
  "ResultURL": "http://myservice:8080/result"
}
```

## Request Parameter Definition

| Name | Description | Type | Sample |
|------|-------------|------|--------|
| InitiatorName | Initiator username for authentication | Alpha-Numeric | testapi772 |
| SecurityCredential | Encrypted initiator password | String | EToK4lNR... |
| CommandID | Command type (SalaryPayment, BusinessPayment, PromotionPayment) | String | SalaryPayment |
| Amount | Top-up amount in KES | Numeric | 500 |
| PartyA | Sender shortcode/till | Numeric | 600000 |
| PartyB | Recipient phone number (format: 2547XXXXXXXX) | Numeric | 254722000000 |
| Remarks | Comments sent with transaction | String | Top up account |
| QueueTimeoutURL | Endpoint for timeout notification | URL | https://ip:port/path |
| ResultURL | Destination URL for result | URL | https://ip:port/path |

## Response Body
```json
{
  "OriginatorConversationID": "515-5258779-3",
  "ConversationID": "AG_20200123_0000417fed8ed666e976",
  "ResponseCode": "0",
  "ResponseDescription": "Accept the service request successfully"
}
```

## Response Parameter Definition

| Name | Description | Type | Sample |
|------|-------------|------|--------|
| OriginatorConversationID | Unique identifier of request message | String | 515-5258779-3 |
| ConversationID | Unique identifier generated by M-Pesa | String | AG_20200123_0000417fed8ed666e976 |
| ResponseCode | Success indicator (0=success) | String | 0 |
| ResponseDescription | Response status description | String | Accept the service request successfully |

## Error Codes

| errorCode | errorMessage | Mitigation | HTTP |
|-----------|-------------|------------|------|
| 400.002.02 | Bad Request - Invalid Phone | Ensure valid phone format | 400 |
| 401.002.01 | Invalid Access Token | Regenerate token | 401 |
| 404.002.01 | Resource not found | Check API endpoint | 404 |
| 500.001.1001 | Amount exceeds limit | Reduce amount requested | 500 |
| 500.003.02 | System is busy | Retry after short wait | 500 |
