# Account Balance API

Enquire the balance on an M-Pesa BuyGoods (Till Number).

**Endpoint:** `POST https://sandbox.safaricom.co.ke/mpesa/accountbalance/v1/query`

## Overview
The M-PESA Account Balance API enables organizations to programmatically check their M-PESA account balances. This is essential for automating financial processes, monitoring account status, and ensuring sufficient funds for operations.

### Key Features
- **Real-time Balance Inquiry:** Instantly retrieve your M-PESA account balance
- **Secure Access:** Authentication and authorization ensure only permitted entities access the balance
- **Automated Responses:** Receive automatic acknowledgments and responses to inquiries

## How It Works
1. **Request Setup and Submission:** Organization configures and sends the request to the endpoint
2. **Acknowledgment by Daraja:** API Management Platform validates, authorizes, and authenticates the request
3. **Forwarding to M-PESA:** Daraja forwards the validated request to M-PESA
4. **Acknowledgment by M-PESA:** M-PESA validates and acknowledges receipt
5. **Internal Processing:** M-PESA processes the request and prepares balance information
6. **Response Callback:** M-PESA sends the balance response to the organization's callback URL
7. **Status Check on Failure:** No automatic retries if callback fails; organization can manually check via M-PESA portal

> **Note:** This API is asynchronous and can be accessed via the internet, VPN, or Multiprotocol Switch.

## Getting Started
### Prerequisites
- Create a Daraja Account on the Safaricom Developer Portal
- Create a sandbox app to obtain API credentials
- Retrieve Consumer Key & Consumer Secret from My Apps
- Use test data from the simulator section
- For production, ensure you have a live pay bill/till number with Business Admin/Manager operators

### Good to Know
This API can check the status of various M-PESA transactions including C2B, B2B, B2C, Reversal, and IMT.

### Authentication
Generate an access token before making API calls (see Authorization API).

## Use Cases
- **Financial Reconciliation:** Schedule balance queries at end of day to reconcile transactions
- **Automated Payment Systems:** Check sufficient funds before initiating vendor payments
- **Fraud Monitoring:** Periodic balance checks to detect unauthorized withdrawals
- **Cash Flow Management:** Monitor multiple branch account balances
- **Payroll Processing:** Confirm funds before salary disbursement
- **Bill Payments:** Track received payments for utility companies

## Request Body
```json
{
  "Initiator": "testapiuser",
  "SecurityCredential": "SAFVNChNHfVtXEZMBuVo+a1Hwr+DtrUVN3zVg==",
  "CommandID": "AccountBalance",

  "IdentifierType": "4",
  "Remarks": "ok",
  "QueueTimeOutURL": "http://myservice:8080/queuetimeouturl",
  "ResultURL": "http://myservice:8080/result"

```

### Request Parameter Definition

| Name | Description | Type | Sample |
|------|-------------|------|--------|
| CommandID | Unique command passed to M-PESA system | String | AccountBalance |
| PartyA | Shortcode of the organization querying balance | Numeric | 600772 |
| IdentifierType | Type of organization | Numeric | 4 |
| Remarks | Comments sent with the transaction | String | tests |
| Initiator | API user credential/username for authentication | Alpha-Numeric | Testapi772 |
| SecurityCredential | Encrypted initiator password | String | EToK4lNR... |
| QueueTimeoutURL | Endpoint for timeout notification | URL | https://ip:port/path |
| ResultURL | Destination URL for result notification | URL | https://ip:port/path |

## M-PESA Organization Portal

### Access
URL: https://org.ke.m-pesa.com/orglogin.action

### First-Time Login
1. Launch `https://org.ke.m-pesa.com`
2. Enter Short code (Bulk payment number)
3. Enter Business Administrator username
4. Enter first-time password (case-sensitive)
5. Enter Verification Code and click login
6. Enter OTP to proceed to change password page
7. Set new password, confirm, and set security questions/answers
8. Submit to activate account

### Account Types
- **MMF/Working/M-PESA Account:** For business withdrawals
- **Utility Account:** Receives customer payments
- **Charges Paid Account:** Debited for transaction charges (always negative)
- **Organization Settlement Account:** Settles charges and moves balance automatically

### Portal Roles

#### Business Administrator
- Creates system users and assigns roles
- Cannot view transactions
- Created by Safaricom

#### Business Manager
- Views statements, initiates transactions, approves/rejects transactions
- Withdraws funds from M-PESA

#### API User Creation
1. Log in as Business Administrator
2. Select operators → Add → Enter API initiator username
3. Select access channel as API
4. Assign API roles, submit KYC info
5. Set password via Business Manager (My Functions → Operator Management)

## FAQs
- **What is the M-PESA Account Balance Query API?** Allows organizations to programmatically check M-PESA account balances
- **Who can use it?** Any organization with an M-PESA account registered for Daraja API access
- **What information is required?** Short code, access token, Command ID (AccountBalance), PartyA, IdentifierType (4), callback URL
- **How does authentication work?** OAuth 2.0 with access token
- **What is the typical response time?** Synchronous acknowledgment within seconds; final balance within minutes
- **What if the callback URL is unreachable?** No automatic retries; manually query transaction status
- **Can I check multiple accounts?** Yes, separate requests for each account
- **Is balance in real-time?** Yes
- **What security measures are in place?** OAuth 2.0, HTTPS encryption
- **Can I automate balance checks?** Yes
- **Can third parties check balance?** No, only own shortcode
